# Требования к системе — PassVault

## 1. Проблема

Пользователи используют слабые или одинаковые пароли, потому что сложные сложно придумать и запомнить. Нужен инструмент, который генерирует надёжный пароль и позволяет хранить его в одном месте.

## 2. Целевая аудитория

Студенты и пользователи, которые:
- хотят использовать уникальные пароли для каждого сайта,
- не пользуются коммерческими менеджерами паролей,
- ищут простое локальное решение.

## 3. Функциональные требования

| # | Требование |
|---|-----------|
| FR-1 | Пользователь может настроить параметры генерации: длина (4–64), наличие заглавных букв, цифр, спецсимволов |
| FR-2 | Система генерирует пароль на лету (без перезагрузки страницы) |
| FR-3 | Пользователь может сохранить пароль с меткой (название сайта) и логином |
| FR-4 | Список паролей отображается со скрытым значением (••••••••) |
| FR-5 | Пользователь может показать/скрыть конкретный пароль |
| FR-6 | Пользователь может скопировать пароль в буфер одной кнопкой |
| FR-7 | Поиск по метке и логину |
| FR-8 | Редактирование существующей записи |
| FR-9 | Удаление записи с подтверждением |

## 4. Нефункциональные требования

- Приложение работает локально на Python 3.11+
- Время ответа < 200 мс для всех страниц
- Данные хранятся в SQLite-файле на диске
- Интерфейс адаптирован для мобильных устройств

## 5. Схема архитектуры

```
Браузер
  │  GET / POST (HTML-формы, Fetch API)
  ▼
Flask (app.py)
  ├─ GET  /            → index()    → index.html
  ├─ POST /generate    → generate() → JSON
  ├─ POST /save        → save()     → redirect
  ├─ GET  /edit/<id>   → edit()     → edit.html
  ├─ POST /edit/<id>   → edit()     → redirect
  └─ POST /delete/<id> → delete()   → redirect
  │
  ▼
database.py (sqlite3)
  └─ passwords.db (SQLite-файл)
       └─ TABLE passwords (id, label, login, password, created_at)
```
