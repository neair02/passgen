# Архитектура — PassVault

## Стек технологий

### Backend: Flask
Выбран Flask, а не Django, потому что:
- Приложение небольшое — нет смысла тащить ORM, admin-панель и всё остальное
- Flask позволяет написать рабочий прототип за один файл `app.py`
- Стандартная библиотека `sqlite3` покрывает все нужды без ORM

### База данных: SQLite
- Не требует отдельного сервера
- Хранится в одном файле `passwords.db` рядом с кодом
- Стандартный модуль Python — никаких доп. зависимостей

### Frontend: Vanilla HTML/CSS/JS
- Нет фреймворков — всё написано вручную для понимания основ
- Один CSS-файл с CSS-переменными
- Fetch API для генерации пароля без перезагрузки страницы

## Структура базы данных

```sql
CREATE TABLE passwords (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    label      TEXT    NOT NULL,        -- название сайта/сервиса
    login      TEXT    DEFAULT '',      -- логин или email
    password   TEXT    NOT NULL,        -- пароль (хранится открытым текстом*)
    created_at DATETIME DEFAULT (datetime('now','localtime'))
);
```

> *В учебном проекте шифрование не реализовано. В боевой системе пароли необходимо шифровать (например, Fernet из библиотеки `cryptography`).

## Поток данных: генерация и сохранение

```
1. Пользователь задаёт параметры и нажимает «Сгенерировать»
2. JS делает POST /generate (Fetch API)  ──►  Flask возвращает JSON {"password": "..."}
3. JS вставляет пароль в поле вывода
4. Пользователь заполняет метку и нажимает «Сохранить»
5. JS кладёт пароль в скрытое поле <input type="hidden">
6. Форма отправляется POST /save  ──►  Flask пишет в SQLite  ──►  redirect /
```

## Модули

| `static/css/style.css` | Все стили |
| `tests/test_app.py` | Автоматические тесты (pytest) |
